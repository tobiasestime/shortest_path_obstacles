Shortest Path on Grid with Obstacles

Implementation by Tobias Estime

 

  1. Click a cell (one or more times) to make it an obstacle
  2. Hold the shift key and click a cell (shift + click) to make it the starting cell
  3. Hold the alt key and click a cell (alt + click) to make it the ending cell
  4. Select a contiguity case to display the shortest path, random to setup the grid somewhat randomly, or clear the grid

Following chess terminology, the rook's contiguity case only considers vertical and horizontal paths - parallel to rows or columns, as a rook moves in a chess game. The queen's contiguity case considers diagonal paths in addition to horizontal and vertical ones. The distance between any two adjacent cells in either case is considered the same in this example.

Path length is defined as the number of cells between starting and ending cells (exclusive). A random shortest path will be chosen if there are ties.
